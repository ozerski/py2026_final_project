### 6. Система управления задачами (Task Tracker)
#### py_adv_student79

* **Цель:** Управление жизненным циклом проектных задач.
* **Функциональные требования:**
* Ведение бэклога задач с атрибутами: статус, приоритет, исполнитель, дедлайн.
* Контроль состояний задачи (валидация переходов между статусами).
* Реализация логики сравнения приоритетов задач.
* Генерация еженедельного таймшита с условным форматированием по критерию просрочки дедлайна.


* **Результат:** Актуальное состояние бэклога в базе данных и отчет по трудозатратам/дедлайнам.

---

**Общие требования к реализации (для всех проектов):**

* **Система контроля версий:** Наличие истории коммитов с описанием изменений.
* **Структура:** Соответствие анатомии профессионального проекта (пакетная структура, конфигурационные файлы).
* **Качество:** Полное покрытие кода аннотациями типов, соответствие стандартам чистого кода.
